# 管理端请求返回 401

## 现象

聊天、知识库或管理接口突然返回 401，前端提示登录过期并跳转登录页。

## 根因

JWT 缺失、已过有效期或签名无效。继续使用旧页面中的确认令牌也无法恢复认证。

## 解决方案

前端会将同一批并发请求产生的 401 合并为一次提示和一次登录页跳转。重新登录获取新 Token，再重新打开原会话。若正在执行待确认操作，重新读取操作状态；已经过期的令牌必须轮换或重新生成预览。登录 Token 默认有效期由 `ACCESS_TOKEN_EXPIRE_MINUTES` 配置，当前为 10080 分钟（7 天）。

## 验证

登录后调用用户信息或 `/api/knowledge/status` 返回 200，聊天请求能建立 SSE 流。
